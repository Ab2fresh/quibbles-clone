How I Executed the Project
Step 1: Creating the Structure
I started with the index.html file and planned out where each section of the page would go.
Created a main container to represent a mobile phone screen.
Added the header section, search bar, tags section, content feed, and bottom navigation bar.
Organized the page into sections to make it easier to style later.
Step 2: Styling with CSS
After building the structure, I used CSS to make the page look more like a real application.
Created color variables to keep the design consistent.
Used Flexbox to align items such as icons and navigation elements.
Used CSS Grid to arrange post images neatly.
Added horizontal scrolling for tags and featured content sections.
Hid scrollbars to improve the overall appearance.
Step 3: Testing and Fixing Issues
While working on the project, I encountered some challenges and learned a lot through trial and error.
Fixed issues with overflowing content.
Corrected grid layouts that were not displaying properly.
Learned how to use browser developer tools to inspect and debug elements.
Worked with positioning and z-index to keep important buttons visible while scrolling.
Step 4: Deployment
To make the project accessible online:
I used Git to keep track of my progress.
Uploaded the project to GitHub.
Connected the repository to Vercel for automatic deployment.
Live Demo: https://quibbles-clone.vercel.app/
What I Learnt
This project helped me improve my understanding of:
HTML page structure
CSS Flexbox and Grid
Responsive design concepts
Handling overflow and scrolling
Basic debugging techniques
Git and GitHub workflow
Deploying projects with Vercel
Although I am still learning, this project gave me practical experience in building and deploying a frontend application
